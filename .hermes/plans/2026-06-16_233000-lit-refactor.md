# Lit 重构计划 — literature-skill 全面重写

> **状态**: 计划稿，待批准
> **目标**: 从"多个入口文件 + papers.json 中间状态"的屎山架构，重构为"Zotero 作数据库、单一 CLI 入口、按功能分包"的干净架构
> **原则**: DRY, YAGNI, Zotero is the source of truth, 不增加功能 scope

---

## 一、现状问题

### 1.1 入口爆炸

| 文件 | 行数 | 职责 | 问题 |
|------|------|------|------|
| `main.py` | 552 | 单篇 PDF 下载 | 和 zot.py 职责重叠 |
| `zot.py` | 662 | 单篇元数据+下载+挂载 | 内部调 main.py，依赖隐式 |
| `people.py` | 1634 | 学者全流程 | 史上最大，混了 scrape/register/download/template + 遗留代码 |
| `zotero_attach.py` | 262 | 批量补 PDF | 新写的，本应替代 people.py 的下载部分 |
| `ocr_citation.py` | 262 | 图片→引用 | 独立入口，和主流程无连接 |
| `pdf_parser.py` | 501 | MinerU PDF 解析 | 独立入口 |
| `generate_qr.py` | 87 | 生成二维码 | 独立入口 |

共 **6 个用户入口 + 3 个被入口**。用户说"帮我把这篇论文加到 Zotero"——你应该用哪个？`zot.py`？`main.py`？`people.py -paper`？鬼知道。

### 1.2 状态文件瘟疫

`people/data/*.json` 是 Scratch 文件，不是数据库。每次操作（scrape → DOI match → register → download）都写一次 JSON，但：
- 用户清洗 Zotero 后 JSON 不同步
- 并发进程互相覆盖 JSON
- `dl_status` 字段和 Zotero 真实状态脱节

### 1.3 配置重复

`load_config()` 抄了 4 遍（people.py, zot.py, zotero_attach.py, pdf_parser.py）。

### 1.4 命名混乱

- `main.py` — 根本不是 main，是「单篇论文下载器」
- `zot.py` — 不是 Zotero 工具库，是「单篇论文全套流程」
- `url_parser.py` — 名字像 URL 解析，实际还管 `resolve_doi()`

## 三、CLI 设计

```bash
lit scholar <URL>              # [discover] 抓取 Scholar → 注册到 Zotero（无 PDF）
lit import <DOI/URL>           # [discover] 单篇→注册 Zotero（无 PDF）
lit import <image_path>        # [discover] 图片 OCR → 注册 Zotero
lit attach <collection>        # [batch] 读 Zotero collection → 批量补 PDF
lit download <DOI/URL>         # [download] 下载 PDF 到本地（不入 Zotero）
lit parse <pdf_path>           # [digest] MinerU 解析 PDF → Markdown
lit digest <collection>        # [digest] 读 Zotero collection → 生成消化报告
lit qr <DOI>                   # [utility] 生成 QR 码
```

**对比现在：**

| 要做什么 | 现在 | 重构后 |
|---------|------|--------|
| 学者全套 | `people.py "URL" --scrape-only` + `--register-only` + `zotero_attach.py --collection` + `--template-only` | `lit scholar <URL>` + `lit attach <name>` + `lit digest <name>` |
| 单篇加 Zotero | `zot.py <URL>` | `lit import <DOI/URL>` |
| 只下载 PDF | `main.py <URL>` | `lit download <DOI/URL>` |
| 批量补 PDF | `zotero_attach.py --collection "X"` | `lit attach <collection>` |
| 解析 PDF | `pdf_parser.py <pdf_path>` | `lit parse <pdf_path>` |

### 3.1 为什么不用子命令？`lit scholar` 而不是 `lit --scholar <URL>`

因为每条命令的 flag 完全不同：
- `lit import <DOI>` — 没有 flag
- `lit scholar <URL> --max-papers 10` — 有 `--max-papers`
- `lit attach <collection> --limit 5` — 有 `--limit`
- `lit digest <collection> --output custom.md` — 有 `--output`

子命令比 `--flag <arg>` 更自然。argparse 的 `add_subparsers()` 原生支持。

### 3.2 依赖 Zotero 的命令

`lit import` / `lit scholar` / `lit attach` / `lit digest` 需要 Zotero 配置。
`lit download` / `lit parse` / `lit qr` 不需要。

```python
"""
lit/core/config.py — 配置加载（单例模式）

所有模块从此处加载配置，不再各自写 load_config()。
"""
from __future__ import annotations
import sys
from pathlib import Path
from typing import Any

import yaml
from rich.console import Console

console = Console()
_SKILL_BASE = Path(__file__).resolve().parent.parent.parent  # lit/../ = skill base
_CONFIG_PATH = _SKILL_BASE / "config.yaml"
_cache: dict | None = None


def skill_base() -> Path:
    """Resolve the skill base directory (parent of lit/)."""
    return _SKILL_BASE


def load() -> dict:
    """Load config.yaml. Cached after first call. Dies with error if missing."""
    global _cache
    if _cache is not None:
        return _cache
    if not _CONFIG_PATH.exists():
        console.print(f"[red]config.yaml not found at {_CONFIG_PATH}[/red]")
        console.print(f"  Copy config.yaml.example → config.yaml and fill in credentials.")
        sys.exit(1)
    with open(_CONFIG_PATH, "r", encoding="utf-8") as f:
        _cache = yaml.safe_load(f) or {}
    return _cache


def zotero() -> dict:
    """Shortcut: get zotero section with validation."""
    cfg = load()
    z = cfg.get("zotero", {})
    for key in ("library_id", "api_key"):
        val = z.get(key, "")
        if not val or val.startswith("YOUR_"):
            console.print(f"[red]zotero.{key} not configured in config.yaml[/red]")
            sys.exit(1)
    return z
```

### Phase 1：CLI 入口（1 步）

#### Step 3: 写 `cli.py` — 单一入口

**做什么**: argparse 子命令调度，每个子命令调对应的模块函数。

```python
"""
lit/cli.py — CLI 入口。

用法:
    lit scholar <URL>
    lit import <DOI/URL>
    lit attach <collection>
    ...
"""
import sys
import argparse


def build_parser() -> argparse.ArgumentParser:
    parser = argparse.ArgumentParser(
        prog="lit",
        description="Literature Skill — 论文管理工具",
    )
    sub = parser.add_subparsers(dest="command", required=True)

    # scholar
    p = sub.add_parser("scholar", help="抓取 Scholar Profile 并注册到 Zotero")
    p.add_argument("url", help="Google Scholar profile URL")
    p.add_argument("--max-papers", type=int, default=None)

    # import (DOI/URL/image)
    p = sub.add_parser("import", help="单篇论文→注册 Zotero（可选下载 PDF）")
    p.add_argument("source", help="DOI, URL, 或图片路径")
    p.add_argument("--download", action="store_true", help="同时下载 PDF")

    # download (only PDF, no Zotero)
    p = sub.add_parser("download", help="下载 PDF 到本地")
    p.add_argument("source", help="DOI 或 URL")

    # attach (batch)
    p = sub.add_parser("attach", help="读 Zotero collection → 批量补 PDF")
    p.add_argument("collection", help="Zotero collection name")
    p.add_argument("--parent", default="People")
    p.add_argument("--limit", type=int, default=None, help="Limit papers")

    # parse (MinerU)
    p = sub.add_parser("parse", help="MinerU 解析 PDF → Markdown")
    p.add_argument("pdf_path", help="PDF 文件路径")
    p.add_argument("-o", "--output", help="输出文件路径")

    # digest
    p = sub.add_parser("digest", help="生成文献消化报告")
    p.add_argument("collection", help="Zotero collection name")
    p.add_argument("--parent", default="People")
    p.add_argument("-o", "--output", help="自定义输出路径")

    # qr
    p = sub.add_parser("qr", help="生成 DOI QR 码")
    p.add_argument("doi", help="DOI")

    return parser


def main():
    parser = build_parser()
    args = parser.parse_args()

    if args.command == "scholar":
        from lit.discover.scholar import run
        run(args.url, max_papers=args.max_papers)

    elif args.command == "import":
        from lit.discover.cite import run
        run(args.source, download=args.download)

    elif args.command == "download":
        from lit.download.engine import download_one
        download_one(args.source)

    elif args.command == "attach":
        from lit.batch.attach import run
        run(args.collection, parent=args.parent, limit=args.limit)

    elif args.command == "parse":
        from lit.digest.parser import run
        run(args.pdf_path, output=args.output)

    elif args.command == "digest":
        from lit.digest.template import run
        run(args.collection, parent=args.parent, output=args.output)

    elif args.command == "qr":
        from lit.generate_qr import run  # 保留原文件
        run(args.doi)


if __name__ == "__main__":
    main()
```

**flag 变化**: 不再需要 `--scholar-name`、`--register-only`、`--template-only`、`--download-only`、`--attach-missing`——collection name / 动作本身就是位置参数。

|------|------|------|
| 重构期间用户想用旧代码 | 高 | 旧文件不动，改成薄封装调新代码。`python main.py` 在重构期间仍可用 |
| pyzotero 接口变化 | 低 | core/zotero.py 做隔离层，只暴露高层接口 |
| 配置路径不对 | 中 | `config.py` 用 `Path(__file__).resolve().parent.parent.parent` 自动解析 skill base |
| 删除 JSON 后 cron 作业崩溃 | 中 | cron 改用 `lit attach`，旧 cron 暂停直到重构完成 |
| 重构过程中有人提交更改 | 低 | 新旧文件共存，git merge 不会有冲突 |
| 《等不及重构完成就要用》 | 中 | 核心功能（scholar → attach → digest）在 Step 9 之后就可用了 |

**回退策略**: 如果重构到一半发现问题，旧入口文件都是薄封装——退回到直接调旧函数也兼容。所有旧文件保持 `import` 可用。

## 九、开放问题

1. **`papers.json` 完全删除还是保留只读备份？** 建议删除——它已经是不可靠状态。但可以留一个 `people/data/ARCHIVED_*.json.old` 作为不可删的备份，3 个月后清理。

2. **`download_monitor.py` 和 `click_engine.py` 是否并入 `download/` 包？** 它们只在特定 publisher 适配器里用，保持原位，不搬。

3. **`zot.py` 的 OCR 图片→引用逻辑怎么处理？** 建议直接放进 `discover/cite.py`——`lit import <image>` 检测到文件是图片就走 OCR 路径。

4. **`digest/template.py` 从 Zotero 读数据时，要不要加上 MinerU 解析？** 按现有设计，digest 就是纯模板生成（不包含深读阶段）。MinerU 解析是单独的 `lit parse`。保持分离。

5. **config.yaml 中 `people` 段的配置（parent_collection, scholar delay 等）怎么迁移？** 直接改为 `lit` 段：

```yaml
lit:
  parent_collection: People
  scholar:
    page_delay: 3000
    min_delay: 5
    max_delay: 15
```

旧 `people` 段兼容读取，新代码优先读 `lit` 段。
