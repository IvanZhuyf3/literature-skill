# WeChat Integration Notes

## hermes-wechat (Native Gateway Adapter)

The recommended WeChat integration for Hermes is the **hermes-wechat** native Gateway adapter (https://github.com/RongleCat/hermes-wechat), which uses the iLink Bot API (微信官方开放协议).

### Setup Summary

1. **Install deps**: `cd ~/.hermes/hermes-agent && uv add aiohttp cryptography qrcode pillow`
2. **Deploy adapter**: `cp references/weixin.py ~/.hermes/hermes-agent/gateway/platforms/weixin.py`
3. **Hermes already has WEIXIN support built-in** (Platform enum, run.py adapter factory) as of recent versions — check before patching.
4. **Login**: Run `qr_login()` from the adapter to get account_id + token. Use foreground + `pty=true` with 120s timeout so QR displays in TUI.
5. **Configure** `~/.hermes/config.yaml`:
   ```yaml
   platforms:
     weixin:
       enabled: true
       extra:
         account_id: "<from login>"
         token: "<from login>"
         base_url: "https://ilinkai.weixin.qq.com"
         dm_policy: "open"
         allow_from: []
       home_channel:
         platform: "weixin"
         chat_id: "<user_id from login>"
   platform_toolsets:
     weixin:
     - hermes-cli
   ```
6. **Enable auth**: Add `GATEWAY_ALLOW_ALL_USERS=true` to `~/.hermes/.env`
7. **Restart**: `hermes gateway restart`

### Capabilities
- Text message send/receive (auto-segment long messages at 4000 chars)
- **Image send/receive** (AES-128-ECB encrypted CDN, automatic decrypt)
- **File send/receive** (PDF, DOC, ZIP, etc., AES decrypt + original filename preserved)
- Video receive, voice-to-text
- QR code terminal login with auto-refresh (3 attempts)
- Quoted message support (text + media/file references)
- Typing indicator
- Message deduplication (5-min sliding window)

### Sending Files/Images to WeChat from Agent
The hermes-wechat adapter supports sending files and images. Use `send_message` targeting `weixin:<chat_id>` with `MEDIA:<local_path>` to send PDFs, QR code PNGs, etc. directly to the user's WeChat.

### Limitations
- No Markdown rendering (WeChat client strips formatting; adapter does strip processing)
- No voice message sending (iLink API limitation)
- No emoji/sticker/GIF sending
- Group chat routing not fully polished (all group messages trigger reply, no @-bot filtering)
- Token expires — need to re-scan QR if session is lost

## QR Code Presentation

For QR login: foreground + `pty=true` with long timeout displays QR art directly in TUI.

For QR code files generated by `generate_qr.py`: the PNG can be sent to WeChat via the adapter's file sending capability.

## Deprecated

- **wechat-acp** (npx wechat-acp): Text-only one-way bridge, no file capability. Replaced by hermes-wechat.
